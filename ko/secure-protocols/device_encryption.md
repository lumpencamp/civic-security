# 장치 암호화: 미사용 데이터의 암호화 방어

*상태: 저장소 암호화 매뉴얼 | 대상: 소스 처리자 및 고위험 데이터 관리자*

전원이 꺼진 상태에서 물리적 장치가 압류되는 경우 FDE(전체 디스크 암호화)가 유일한 방어선입니다. 암호화 알고리즘이 "깨지는" 경우는 거의 없습니다. 실패 지점은 거의 항상 약한 키 생성(암호 엔트로피) 또는 결함 있는 구현입니다.

이 가이드에서는 표적 포렌식 추출 및 GPU 가속 무차별 공격을 견디는 데 필요한 FDE 및 거부 가능한 암호화 아키텍처의 배포에 대해 자세히 설명합니다.

---

## 1. 암호화 기본 요소 및 엔트로피 요구 사항

데이터를 보호하는 수학적인 방법을 이해하지 못한 채 기본 설정에 의존하지 마십시오.

### 권장 기본 요소
암호화 소프트웨어(예: VeraCrypt 또는 LUKS)를 구성할 때 알고리즘을 선택하라는 메시지가 표시됩니다.
* **암호화 알고리즘:** **AES-256**을 사용합니다. 이는 철저한 감사를 거쳐 최신 CPU(AES-NI)에서 하드웨어 가속되는 글로벌 표준으로, 보안을 손상시키지 않으면서 빠른 읽기/쓰기 속도를 제공합니다. 암호화 모드는 블록 스토리지에 대한 조작 공격에 저항하도록 특별히 설계된 **XTS**(암호문 도용이 포함된 XEX 기반 조정 코드북 모드)여야 합니다.
* **해시 알고리즘:** **SHA-512** 또는 **Whirlpool**을 사용합니다. 이는 키 파생(비밀번호를 실제 수학 키로 전환)에 사용됩니다.

### GPU 무차별 대입에 대한 암호 엔트로피
법의학 공격자(T3/T4)는 대규모 GPU 클러스터를 사용하여 초당 수백만의 속도로 암호를 추측합니다.
* **명령:** 귀하의 암호는 우주가 열사하기 전에 수학적으로 무차별 대입을 불가능하게 만들 만큼 충분한 엔트로피(무작위성)를 가져야 합니다.
* **구현:** 최소 **완전히 임의의 단어 6개**로 구성된 **Diceware** 암호를 사용합니다(예: '올바른 말 배터리 스테이플...'). 이는 77비트 이상의 엔트로피를 생성하며, 이는 알려진 모든 기존 무차별 대입 공격에 대해 안전합니다. 대체 문자가 포함된 복잡하고 기억된 비밀번호(예: `P@ssw0rd!123`)를 사용하지 *마세요*. 크래킹 사전은 이러한 사전을 즉시 물리칩니다.

## 2. 전체 디스크 암호화(FDE) 배포

FDE는 암호화되지 않은 민감한 데이터 조각이 포함되어 있는 스왑 공간과 임시 파일을 포함하여 드라이브의 모든 섹터를 암호화합니다.

### 리눅스(LUKS)
* **구현:** Linux 통합 키 설정(LUKS)은 Linux 배포판의 표준입니다. 보안 Linux OS(Debian, Ubuntu, Qubes OS 등)를 설치하는 동안 설치를 암호화하라는 메시지가 표시됩니다.
* **보안 표준:** 설치 프로그램이 LUKS2를 사용하는지 확인하세요. LUKS2는 Argon2id KDF(키 파생 함수)를 활용합니다. Argon2id는 특히 메모리 하드형으로 설계되어 적이 GPU 또는 ASIC을 사용하여 암호 문구를 무차별 공격하는 데 필요한 재정적 비용과 시간을 기하급수적으로 증가시킵니다.

### 크로스 플랫폼(VeraCrypt)
외장 하드 드라이브, USB 썸 드라이브 또는 Windows/macOS에서 암호화된 컨테이너 생성의 경우 **VeraCrypt**(TrueCrypt의 감사된 오픈 소스 후속 제품)를 사용하세요.

## 3. 거부 가능한 암호화: VeraCrypt 숨겨진 볼륨

신체적 강압이나 법적 강제(암호 문구를 요구하는 법원 명령)에 직면한 경우 키를 강제로 넘겨주어야 하기 때문에 표준 FDE가 실패합니다. 거부 가능한 암호화는 동일한 공간에 외부 볼륨과 숨겨진 볼륨이라는 두 개의 볼륨을 생성하여 이 문제를 해결합니다.

### 타당한 부인 가능성의 메커니즘
VeraCrypt는 미끼 데이터로 채워진 외부 볼륨을 생성합니다. 해당 외부 볼륨의 무작위로 보이는 "여유 공간" 내부에 숨겨진 볼륨이 구축됩니다. 숨겨진 볼륨이 존재함을 증명하는 암호화 서명은 없습니다.

암호를 강제로 제공해야 하는 경우 외부 볼륨에 대한 암호를 제공합니다. 공격자는 드라이브의 암호를 해독하고 미끼 데이터를 찾았지만 내부에 보조 데이터가 숨겨져 있다는 수학적 증거는 없습니다.

### 구성 프로토콜
1. VeraCrypt를 열고 **볼륨 생성**을 선택합니다.
2. **숨겨진 VeraCrypt 볼륨 생성**을 선택합니다.
3. **외부 볼륨 생성:**
    * 외부 볼륨 암호를 설정합니다.
    * 마운트하고 그럴듯한 미끼 데이터(예: 세금 서류, 양성 개인 사진)로 채웁니다. 미끼 데이터는 *반드시* 현실적으로 보여야 합니다.
4. **숨겨진 볼륨 생성:**
    * VeraCrypt는 이제 외부 볼륨의 여유 공간 내에 숨겨진 볼륨을 생성하라는 메시지를 표시합니다.
    * 숨겨진 볼륨에 대해 완전히 다른 매우 안전한 Diceware 암호를 설정하십시오.
5. **행동 제약(중요):** 더 많은 미끼 데이터를 추가하기 위해 외부 볼륨을 마운트할 때 *반드시* 마운트 옵션에서 "숨겨진 볼륨 보호"를 선택하고 숨겨진 볼륨의 비밀번호를 입력해야 합니다. 이 작업을 수행하지 못한 경우 외부 볼륨에 데이터를 추가하면 숨겨진 볼륨을 덮어쓰고 영구적으로 파괴됩니다.
