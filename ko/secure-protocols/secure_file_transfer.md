# 안전한 파일 전송: 유출 및 수집 인프라

*상태: 인프라 관리 매뉴얼 | 대상: 소스 처리자, 언론인 및 기술 운영자*

매우 민감한 문서(누출, 증거, 원본 영상)의 유출 및 수집은 모든 작업에서 가장 위험한 단계입니다. 표준 클라우드 서비스(Google Drive, Dropbox)는 중앙 집중식 메타데이터 로깅 및 소환장 준수로 인해 완전히 손상되었습니다.

모든 파일 전송은 IP 공간 및 발신자/수신자 상관 관계를 제거하도록 설계된 분산형 또는 물리적으로 격리된 아키텍처를 통해 발생합니다.

---

## 1. 로컬, 오프라인 공유(시위 및 군중용)

**사용 사례:** 당신은 시위나 행동에 나섰습니다. 셀 서비스가 막히거나 종료되거나 경찰이 Stingrays(IMSI 포착 장치)를 사용하여 네트워크를 모니터링하고 있다고 의심됩니다. 사진, 동영상, 문서를 옆에 있는 동호회 회원과 빠르게 공유해야 합니다.

**작동 방식:** 로컬 장치 간 연결(블루투스 또는 임시 Wi-Fi 핫스팟 생성)을 사용하여 인터넷에 연결하지 않고도 파일을 전송하는 도구를 사용합니다.

* **LocalSend(강력 권장):** LocalSend는 iOS, Android, macOS, Windows 및 Linux에서 사용할 수 있는 무료 오픈 소스 앱입니다. 인터넷 연결이나 계정이 필요하지 않습니다. 두 장치 모두 동일한 로컬 Wi-Fi 네트워크에 있어야 합니다. 한 장치는 모바일 핫스팟을 생성할 수 있고 다른 장치는 셀룰러 데이터를 켜지 않은 상태에서 이에 연결됩니다. 빠르고 안전합니다.
* **Briar:** Briar는 주로 메시징 앱이지만 Bluetooth를 통해 P2P로 파일을 전송할 수도 있습니다. LocalSend보다 느리지만 Wi-Fi 설정이 필요하지 않습니다.
* **AirDrop/Nearby Share에 대한 경고:** Apple의 AirDrop 및 Android의 Nearby Share는 편리하지만 알려진 개인 정보 보호 결함이 있습니다(기기 이름, 때로는 전화 번호/Apple ID를 근처 스캐너에 브로드캐스팅함). 사용하실 경우 "연락처만"으로 설정하시고 전송이 완료되면 즉시 완전히 꺼주세요. 절대로 '모두'에 두지 마세요.

## 2. OnionShare: 임시 P2P 라우팅

**사용 사례:** 어느 쪽도 자신의 IP 주소를 공개하기를 원하지 않으며 중개 서버를 신뢰할 수 없는 지리적 거리에 걸친 직접 일대일 전송입니다.

**역학:** OnionShare는 일시적으로 컴퓨터에서 현지화된 웹 서버를 가동하고 이를 Tor V3 Onion Service에 바인딩합니다. 파일은 수신자가 Tor 네트워크를 통해 직접 다운로드할 때까지 컴퓨터에서 절대 떠나지 않습니다.

### 운영 실행
1. **배포:** OnionShare를 열고 **파일 공유**를 선택합니다. 운영 자산을 로드합니다.
2. **구성:** **파일 전송 후 공유 중지**가 선택되어 있는지 확인하세요. 이렇게 하면 전송이 완료되는 순간 서비스가 자체 파괴되어 재생 또는 검색 공격을 방지할 수 있습니다.
3. **주소 생성:** OnionShare는 무작위 '.onion' 주소와 인증 개인 키를 생성합니다.
4. **OOB(대역 외) 확인:** '.onion' 링크와 비밀번호는 **절대** 전송을 협상한 동일한 채널을 통해 전송되어서는 안 됩니다. Signal을 통한 전송에 동의한 경우 PGP로 암호화된 이메일이나 보안 Matrix 채널을 통해 OnionShare 링크를 보내세요. 이렇게 하면 한 채널을 침해하는 공격자가 인텐트와 페이로드를 모두 가로채는 것을 방지할 수 있습니다.
5. **해지:** 수신자가 수신을 확인하면 OnionShare를 종료합니다. '.onion' 주소는 더 이상 존재하지 않습니다.

## 3. SecureDrop: 기관 수집 아키텍처

**사용 사례:** 조직(NGO, 뉴스룸, 시민 방어 단체)에는 소스를 손상시키거나 조직의 내부 네트워크를 감염시키지 않고 익명의 유출을 수신할 수 있는 지속적이고 고도로 안전한 방법이 필요합니다.

**역학:** SecureDrop은 단일 앱이 아닙니다. 이는 Tor를 통해 실행되는 복잡하고 하드웨어 격리된 네트워크 아키텍처입니다. 제출된 문서에 포함된 제로데이 악성 코드를 차단하기 위해 네트워크 수집 단계와 보기 단계를 물리적으로 분리합니다.

### 4계층 하드웨어 격리 규칙

익명 파일을 안전하게 수집하려면 조직은 다음과 같은 물리적 아키텍처를 배포해야 합니다.

1. **애플리케이션 서버(랜딩 페이지):**
    * *기능:* 이 서버는 Tor 네트워크를 향하고 있습니다. 소스가 문서를 업로드하는 '.onion' 사이트를 호스팅합니다. 안전한 양방향 통신을 허용하기 위해 소스에 대한 고유한 암호화 "코드명"을 생성합니다.
    * *보안:* 조직의 공개 PGP 키를 사용하여 들어오는 문서를 즉시 암호화합니다. 이를 해독할 수 있는 개인 키가 없습니다.
2. **모니터 서버:**
    * *기능:* 침입 시도나 비정상적인 동작이 있는지 애플리케이션 서버를 지속적으로 모니터링합니다.
3. **보안 관측소(SVS) - [Air-Gapped]:**
    * *기능:* 읽기 전용 USB 드라이브에서 Tails OS를 실행하는 물리적으로 에어갭이 있는 전용 노트북입니다. 문서를 해독하는 데 필요한 개인 PGP 키를 보유하고 있습니다. **절대** 인터넷에 연결되지 않습니다.
    * *프로토콜:* 기술 운영자는 암호화된 파일을 애플리케이션 서버에서 암호화된 "전송 USB"로 다운로드합니다. 전송 USB를 SVS에 물리적으로 운반합니다. 파일의 암호가 해독되어 SVS에서 표시됩니다. 파일에 집에 전화를 걸거나 데이터를 유출하도록 설계된 제로데이 악성 코드가 포함되어 있으면 SVS에 네트워크 하드웨어가 없기 때문에 실패합니다.
4. **수출 기지:**
    * *기능:* 문서가 안전하고 게시에 필요한 것으로 확인되면 SVS에서 메타데이터를 대폭 삭제(MAT2 또는 Dangerzone과 같은 도구 사용)하고 깨끗한 내보내기 USB로 이동한 후 표준 작동 시스템으로 전송합니다.

*지침: 인터넷에 연결된 워크스테이션에서 SecureDrop 제출물을 다운로드하고 해독하여 에어갭 프로토콜을 우회하는 것은 조직 보안에 대한 터미널 위반입니다.*
