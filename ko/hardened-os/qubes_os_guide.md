# Qubes OS 아키텍처: Xen 하이퍼바이저 격리

*상태: 엔터프라이즈 아키텍처 매뉴얼 | 대상: 인프라 계획자 및 고위험 대상*

## 데스크탑 OS 선택에 대한 참고 사항

* **초보자용(Linux Mint/Ubuntu):** 현재 Windows 또는 macOS를 사용 중이고 가파른 학습 곡선 없이 중요한 개인 정보 보호 업그레이드를 원한다면 먼저 **Linux Mint** 또는 **Ubuntu**와 같은 사용자 친화적인 Linux 배포판으로 전환하세요. 상용 운영 체제보다 훨씬 덜 침해적입니다.
* **높은 보안(Qubes & Tails)의 경우:** 위협 모델에 국가 수준의 적 또는 고급 감시가 포함되는 경우 Qubes OS(일상적이고 구분된 작업용) 또는 Tails OS(임시적 익명 작업용)를 사용하세요.

---

Qubes OS는 소프트웨어 샌드박싱에 의존하지 않습니다. Xen 하이퍼바이저를 활용한 베어메탈 구획화를 통해 보안을 강화합니다. 단일 애플리케이션이나 도메인이 제로데이 공격으로 인해 손상된 경우 하이퍼바이저는 감염이 하이퍼바이저 경계를 침범하여 다른 도메인이나 호스트 하드웨어를 손상시킬 수 없도록 보장합니다.

이 매뉴얼은 시민 운영에 최적화된 강화된 다중 계층 Xen 도메인 인프라의 단계별 구성을 자세히 설명합니다.

## 1. 네트워크 지형: 엣지 도메인

네트워크 스택은 역사적으로 가장 취약한 공격 표면입니다. Qubes는 네트워크 하드웨어를 방화벽에서, 방화벽을 운영 애플리케이션에서 완전히 격리하여 이를 완화합니다.

* **`sys-net`(하드웨어 인터페이스):** 이 AppVM에는 Wi-Fi 및 이더넷 컨트롤러에 대한 독점적인 PCI 패스스루 액세스가 있습니다. 본질적으로 신뢰할 수 없습니다. 악의적인 Wi-Fi 드라이버가 네트워크 카드를 악용하는 경우 공격자는 개인 데이터가 아닌 'sys-net'에만 액세스할 수 있습니다.
* **`sys-firewall`(트래픽 컨트롤러):** 이 AppVM은 `sys-net` 뒤에 위치합니다. 여기에는 사용자 데이터가 포함되어 있지 않으며 하드웨어에 직접 액세스할 수 없습니다. 이는 'sys-net'에 연결하도록 허용되는 운영 VM을 관리하는 'iptables' 규칙을 엄격하게 적용합니다.

## 2. 익명 라우팅: Whonix 게이트웨이/워크스테이션 쌍

고위험 OSINT 연구 또는 내부 고발자 통신의 경우 표준 VPN으로는 충분하지 않습니다. Whonix 템플릿 쌍을 활용하여 Tor 강제 아키텍처를 구현해야 합니다.

1. **`sys-whonix`(게이트웨이):** 이 ServiceVM은 `sys-firewall`에 직접 연결됩니다. 유일한 암호화 기능은 들어오는 모든 트래픽을 Tor 네트워크를 통해 강제하는 것입니다. 트래픽을 생성하는 사용자 애플리케이션에 대해 전혀 무지합니다.
2. **`anon-whonix`(워크스테이션):** 이 AppVM은 `sys-whonix`에만 *만* 연결합니다. 여기에는 Tor 브라우저와 운영 파일이 포함되어 있습니다. 'sys-firewall' 또는 'sys-net'에 직접 연결되어 있지 않기 때문에 'anon-whonix' 내부를 공격하여 실제 IP 주소를 유출하는 것은 물리적으로 불가능합니다.

## 3. 폐기 가능한 아키텍처: 신뢰할 수 없는 DispVM

영구 작동 VM에서 확인되지 않은 소스에서 전송된 신뢰할 수 없는 첨부 파일(PDF, Word Doc, 이미지)을 절대 열지 마십시오.

* **배포:** DispVM(일회용 가상 머신) 템플릿을 생성합니다. 신뢰할 수 없는 VM에서 다운로드한 파일을 열 때 마우스 오른쪽 버튼을 클릭하고 **DispVM에서 보기**를 선택합니다.
* **역학:** Xen 하이퍼바이저는 RAM에서 격리된 새로운 VM을 인스턴스화하고, 문서를 열고, 창을 닫는 순간 전체 VM을 삭제합니다. 내장된 모든 맬웨어 또는 추적 픽셀은 DispVM과 함께 즉시 제거됩니다.

## 4. Air-Gapped Vault: 루트 키 관리

마스터 PGP 키, 암호화폐 시드 및 비밀번호 데이터베이스는 인터넷에 연결된 도메인에 절대로 접근해서는 안 됩니다.

* **`vault`(오프라인 엔클레이브):** 최소 Debian 또는 Fedora 템플릿을 기반으로 AppVM을 생성합니다.
* **구성:** VM 설정에서 **NetVM**을 '없음'으로 설정합니다. 그러면 가상 이더넷 케이블이 끊어집니다.
* **사용법:** PGP 키를 생성하고 KeePassXC 데이터베이스를 이 저장소에만 저장하세요. 암호화된 데이터는 볼트 *밖으로* 전달되지만 개인 키는 절대로 떠나지 않습니다.

## 5. VM 간 프로토콜 보안(`qvm-copy` / `qvm-move`)

하이퍼바이저는 도메인을 완전히 격리하므로 파일을 끌어서 놓거나 표준 공유 클립보드를 사용할 수 없습니다. 이렇게 하면 맬웨어가 시스템 전체에서 측면으로 마이그레이션되는 것을 방지할 수 있습니다.

* **클립보드 프로토콜:** `vault`에서 `personal`의 웹 브라우저로 비밀번호를 복사하려면:
    1. `vault`에서 텍스트를 강조표시하고 `Ctrl+C`를 누릅니다.
    2. 'Ctrl+Shift+C'를 눌러 하이퍼바이저에게 데이터를 전역 클립보드로 안전하게 이동하도록 지시합니다.
    3. '개인' VM 창을 선택하고 'Ctrl+Shift+V'를 눌러 전역 클립보드에서 데이터를 가져옵니다.
    4. 'Ctrl+V'를 눌러 텍스트를 브라우저에 붙여넣습니다.
* **파일 전송 프로토콜:** 암호화된 문서를 'work'에서 'sys-whonix'로 이동하여 전송하려면:
    1. Dom0 터미널을 열거나 GUI 파일 관리자를 사용하십시오.
    2. `qvm-copy-to-vm [DestinationVM] [파일 이름]`(예: `qvm-copy-to-vm anon-whonix 매니페스트.pdf`)을 실행합니다.
    3. 하이퍼바이저는 파일을 가로채서 명시적인 인증을 요청한 후 대상 VM의 'QubesIncoming' 디렉터리에 안전하게 삽입합니다.
