# Tails OS: 기억 상실 작전 및 휘발성 메모리 보안

*상태: 포렌식 대책 매뉴얼 | 대상: 내부 고발자 및 고위험 연구원*

Tails(Amnesic Incognito Live System)는 일일 운전자가 아닙니다. 전술적 배치 환경입니다. 주요 유틸리티는 단순한 암호화가 아니라 **기억상실**입니다. 호스트 시스템에 흔적이 남지 않습니다.

이 가이드에서는 Tails를 안전하게 배포하는 데 필요한 정확한 운영 제약 조건과 하드웨어 감사에 대해 자세히 설명합니다.

---

## 1. 호스트 컴퓨터 감사(BIOS/UEFI 보안)

Tails는 전적으로 RAM에서 실행됩니다. 그러나 로드되기 전에 호스트 컴퓨터의 펌웨어가 부팅 프로세스를 제어합니다. 호스트 하드웨어가 실제 환경을 손상시키지 않는지 확인해야 합니다.

### 부팅 순서 강화
1. **UEFI/BIOS에 들어갑니다.** 호스트 시스템의 전원을 켜고 설정 키(일반적으로 F2, F12, Delete 또는 Esc)를 반복해서 누릅니다.
2. **보안 부팅 비활성화:** 맬웨어를 차단하도록 설계되었지만 보안 부팅은 라이브 Linux USB를 차단하는 경우가 많습니다. 작업을 위해 일시적으로 비활성화합니다.
3. **빠른 부팅 비활성화:** 빠른 부팅은 하드웨어 초기화 확인을 건너뛰고 USB 감지를 방해할 수 있습니다. 비활성화하십시오.
4. **관리자 비밀번호:** BIOS/UEFI 관리자 비밀번호를 설정합니다. 이렇게 하면 머신이 방치된 경우 Tails가 로드되기 전에 공격자가 악성 USB에서 부팅하기 위해 부팅 순서를 변경하는 것을 방지할 수 있습니다.

## 2. 콜드 부팅 공격 및 RAM 완화

컴퓨터를 종료하더라도 RAM(Random Access Memory)에 저장된 데이터는 즉시 사라지지 않습니다. 법의학 조사관은 RAM 모듈을 동결(압축 공기 또는 액체 질소 사용)하여 특수 기계로 전송하고 성능이 저하되기 전에 암호화 키를 추출하여 "콜드 부팅 공격"을 수행할 수 있습니다.

### 전원 차단 프로토콜
Tails는 종료 프로세스 중에 대부분의 RAM을 덮어쓰도록 설계되었지만 이를 올바르게 트리거해야 합니다.
* **하드 리셋을 수행하지 마십시오:** 단순히 전원 버튼을 누르지 마십시오. 이렇게 하면 RAM이 그대로 유지됩니다.
* **안전한 종료:** 항상 시스템 메뉴를 통해 Tails를 종료하세요.
* **긴급 당김:** 작업 중에 물리적으로 손상이 발생한 경우 USB 스틱을 컴퓨터에서 빠르게 빼냅니다. Tails는 제거를 즉시 감지하고 긴급 메모리 삭제를 실행하며 전원을 끄도록 설계되었습니다.

## 3. LUKS 영구 스토리지: 구성 및 위험

기억소거 시스템은 모든 것을 잊어버립니다. 여러 세션에 걸쳐 파일을 저장해야 하는 경우 영구 스토리지 볼륨을 구성해야 합니다. 이로 인해 엄청난 위험이 발생합니다. 손상되면 볼륨에 변경할 수 없는 작업 기록이 포함됩니다.

### 보안 구성
1. **애플리케이션** > **Tails** > **영구 스토리지**를 엽니다.
2. Tails는 암호화를 위해 LUKS(Linux Unified Key Setup)를 활용합니다.
3. **암호 엔트로피:** 볼륨은 6개 이상의 임의 단어(다이스웨어)로 구성된 암호로 보호되어야 합니다.
4. *법의학 경고:* 이전 버전의 기본 LUKS 반복 횟수는 GPU 무차별 공격에 취약했습니다. Argon2id 키 파생을 활용하여 무차별 대입 공격에 필요한 시간을 기하급수적으로 늘리는 최신 버전의 Tails를 실행하고 있는지 확인하세요.

## 4. "교량 건설" 함정 피하기

활동가들이 Tails 세션을 타협하는 가장 일반적인 방법은 행동 교차를 통해 기억 상실 정체성과 실제 정체성 사이에 "다리"를 만드는 것입니다.

### 운영상의 제약
* **홈 Wi-Fi에 연결하지 않음:** Tails는 모든 트래픽을 Tor를 통해 라우팅하지만 ISP와 Tor 항목 노드는 특정 시간에 홈 IP 주소에서 Tor 연결이 설정된 것을 계속 볼 수 있습니다. 유출된 문서를 업로드하는 경우, 공격자는 업로드 시간과 거주지의 Tor 연결 시간을 연관시킬 수 있습니다. **Tails 작업에는 항상 신뢰할 수 없는 공용 Wi-Fi를 사용하십시오.**
* **단일 세션 규칙:** 동일한 Tails 세션 중에 개인 계정(예: 실제 Gmail)과 운영 계정(예: 익명 ProtonMail)에 로그인하지 마십시오. Tor를 통해서도 종료 노드 또는 고급 지문 채취 기술을 통해 두 세션을 연결하여 운영 별칭을 완전히 태울 수 있습니다.
* **하드웨어 스푸핑:** Tails는 자동으로 네트워크 카드의 MAC 주소를 스푸핑하여 공용 Wi-Fi 라우터에서 하드웨어 ID를 숨깁니다. 종속 포털에 연결하기 위해 명시적으로 요구되지 않는 한 시작 화면에서 **이 기능을 비활성화하지 마십시오**.
