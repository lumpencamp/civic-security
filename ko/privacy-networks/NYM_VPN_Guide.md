# NYM Mixnet: 글로벌 트래픽 분석의 패배

*상태: 분산 암호화 매뉴얼 | 대상: 고위험 운영자 및 네트워크 설계자*

Tor와 같은 표준 익명 네트워크는 공간 난독화(여러 지리적 노드를 통해 트래픽 라우팅)에 의존합니다. 그러나 Tor는 *시간* 또는 *볼륨*을 난독화하지 않습니다. 국가 정보 기관과 같은 GPA(Global Passive Adversary)가 Tor 네트워크의 진입점과 출구점을 동시에 모니터링할 수 있는 경우 타이밍 상관 공격을 수행할 수 있습니다. 네트워크로 들어오는 패킷의 정확한 크기와 타이밍을 네트워크에서 나가는 패킷과 일치시켜 사용자의 익명성을 해제할 수 있습니다.

**NYM Mixnet**은 타이밍 상관 공격을 방지하기 위해 특별히 설계된 차세대 아키텍처입니다.

---

## 1. Tor/VPN과 진정한 믹스넷 아키텍처

NYM이 네트워크 개인 정보 보호의 패러다임 전환인 이유를 이해해야 합니다.

* **표준 3홉 회로(Tor/VPN):** 선입선출(FIFO). 패킷은 가능한 한 빨리 노드를 통과합니다. 기본 방어는 다계층 암호화입니다.
* **믹스넷(NYM):** NYM은 단순히 데이터를 암호화하는 것이 아닙니다. 트래픽 자체의 메타데이터(타이밍, 볼륨 및 순서)를 적극적으로 변경합니다.

### NYM의 메커니즘
NYM은 세 가지 핵심 암호화 혁신을 통해 트래픽 분석을 극복합니다.

1. **스핑크스 패킷 형식:** NYM을 통해 전송된 모든 단일 패킷은 정확히 동일한 크기가 되도록 암호화 방식으로 패딩됩니다. 네트워크를 관찰하는 공격자는 모든 패킷이 동일해 보이기 때문에 큰 이미지 다운로드와 작은 텍스트 메시지를 구별할 수 없습니다.
2. **가변 타이밍 지연:** 패킷이 NYM "혼합 노드"에 진입하면 즉시 전달되지 않습니다. 노드는 무작위로 암호화되어 결정된 1초 동안 패킷을 보유합니다.
3. **더미 트래픽(커버 트래픽):** 데이터를 전송하지 않는 경우 NYM 클라이언트는 암호화된 가짜 "더미" 패킷을 자동으로 생성하여 네트워크로 전송합니다. 이렇게 하면 지속적이고 안정적인 소음 흐름이 보장됩니다.

**결과:** 패킷은 한 순서로 노드에 들어가고, 더미 트래픽과 혼합되고, 무작위로 지연되고, 완전히 다른 순서로 종료됩니다. 글로벌 수동적 공격자는 수신 트래픽과 송신 트래픽을 연관시킬 수 없습니다.

## 2. CLI 배포: NYM 클라이언트 데몬

전술적 작전의 경우 GUI에 의존하지 마십시오. 엄격하고 감사 가능한 라우팅을 보장하려면 명령줄 인터페이스(CLI)를 통해 NYM 클라이언트 데몬을 초기화해야 합니다.

### 1단계: 초기화
귀하의 아키텍처에 맞게 사전 컴파일된 `nym-client` 바이너리를 다운로드하세요. 클라이언트를 초기화하여 암호화 ID 및 로컬 구성 파일을 생성합니다.

```bash
./nym-client init --id OpSec_Alpha
```
*This command generates a unique local ID (`OpSec_Alpha`) and provisions your cryptographic keys.*

### 2단계: 데몬 실행
클라이언트 데몬을 시작합니다. 그러면 NYM 네트워크에 연결되고, 현재 네트워크 토폴로지를 다운로드하고, 로컬 SOCKS5 또는 WebSocket 프록시 수신 포트가 설정됩니다.

```bash
./nym-client run --id OpSec_Alpha
```
*The daemon will output a listening address, typically `127.0.0.1:1977`.*

## 3. 애플리케이션 트래픽 라우팅

NYM 데몬이 활성화되면 애플리케이션이 로컬 프록시를 통해 트래픽을 라우팅하도록 강제해야 합니다.

### SOCKS5 통합
애플리케이션이 SOCKS5 프록시 구성(예: 표준 보안 메시징 클라이언트, 암호화폐 지갑 또는 웹 브라우저)을 지원하는 경우:

1. 애플리케이션의 네트워크 또는 프록시 설정을 엽니다.
2. 프록시 유형을 **SOCKS5**로 설정합니다.
3. 호스트/IP를 '127.0.0.1'로 설정합니다.
4. 포트를 NYM 데몬이 제공하는 포트로 설정합니다(예: 클라이언트 반복에 따라 '1977' 또는 '9000').
5. *중요:* DNS 유출을 방지하려면 "SOCKS v5 사용 시 프록시 DNS"가 선택되어 있는지 확인하세요.

**작동 경고:** NYM은 비동기식 믹스넷입니다. 익명성을 구축하기 위해 의도적으로 패킷을 지연시키기 때문에 **지연 시간이 매우 길다**는 특징이 있습니다. 스트리밍 비디오, VoIP 또는 라이브 웹 브라우징에는 전혀 적합하지 않습니다. NYM은 개인 정보 보호가 속도보다 훨씬 중요한 비동기식 보안 수준이 높은 메시징(예: 현지화된 Matrix 브리지) 및 암호화폐 거래를 위해 설계되었습니다.
